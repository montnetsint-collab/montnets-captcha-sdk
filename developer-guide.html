Montnets Captcha — Developer Documentation

Sliding-puzzle captcha service for B2B integration. Protect your login flows, registration, and sensitive operations against automated bots with invisible-to-the-eye behavioral analysis.

Overview

The Montnets Captcha service provides a sliding-puzzle challenge that combines visual verification with behavioral trajectory analysis. It is designed as a server-side-verified captcha, meaning the final judgment is always performed on your backend — never in the browser or app.

Quick Start

The minimum integration requires 5 steps:

1
Obtain credentials
Log in to the Enterprise Management Console to find your appId and appSecret under your application settings.
2
Create a backend token endpoint
Build a lightweight endpoint on your server (e.g. GET /api/captcha-token). It calls POST /captcha/init with your appId + appSecret and returns only the accessToken to the client.
3
Integrate the SDK
Add the SDK to your Web page, Android app, or iOS app. Provide a tokenProvider that calls your backend endpoint from Step 2.
4
Receive the captchaToken
When the user passes the challenge, the SDK calls your onSuccess callback with a captchaToken. Forward this token to your backend along with the login credentials.
5
Verify server-side
Your backend calls POST /captcha/token/check with appId + captchaToken. If valid=true, proceed with the business logic (login, registration, etc.).

Security Design

ThreatMitigation
Replay attack on /initTimestamp window ±5 min — requests outside the window are rejected with 4002.
Challenge replay / double-submitChallenge encryptedX is atomically retrieved and deleted (GETDEL) on first submit — subsequent attempts fail with CHALLENGE_INVALID.
Token predictionAll tokens (accessToken, captchaToken) are generated with SecureRandom, 36-char base-62, ≥190 bits of entropy.
X-coordinate tamperingThe target X position is stored server-side only and never returned to the client. Clients submit their measured endX; the server compares it against the stored value atomically.
Concurrent token consumptionToken check uses a Lua script executed atomically in Redis — no race conditions are possible.
appSecret exposureappSecret is used only server-to-server (/captcha/init). SDKs use only accessToken.

API Reference

POST /captcha/init
Obtain an Access Token
Called by your backend. Exchanges appId + appSecret for an accessToken that the client SDK will use when loading challenges. Never call this endpoint from browser or mobile app code.

Request body

FieldTypeRequiredDescription
appIdstringRequiredYour application identifier. Available in the Enterprise Management Console.
appSecretstringRequiredYour application secret. Available in the Enterprise Management Console. Keep this on your server only — never expose it in client-side code.
timestamplongRequiredCurrent Unix timestamp in milliseconds. Must be within ±5 minutes of server time.
noncestringRequiredA random string for this request. Used for audit logging; not validated for uniqueness.

Response — data object

FieldTypeDescription
accessTokenstring36-char alphanumeric token. Pass this to the SDK's tokenProvider.
expiresInintegerToken lifetime in seconds. Always 7200 (2 hours).
POST /captcha/init
// Node.js backend example
const res = await fetch('https://captcha-api.int-montnets.com/captcha/init', {
  method:  'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    appId:     process.env.CAPTCHA_APP_ID,
    appSecret: process.env.CAPTCHA_APP_SECRET,
    timestamp: Date.now(),
    nonce:     Math.random().toString(36).slice(2)
  })
});
const json = await res.json();
// json.data.accessToken — return this to the client

POST /captcha/challenge/submit
Submit Slider Result
Called automatically by the SDK after the user releases the slider. The service decrypts the target position, checks the coordinate, and runs the behavioral risk engine. Returns a captchaToken if riskScore ≥ 60.

Request body

FieldTypeRequiredDescription
challengeIdstringRequiredThe challengeId received from /captcha/challenge.
endXintegerRequiredFinal X-coordinate of the slider in natural image pixels.
trajectoryarrayRequiredOrdered list of pointer positions recorded during the drag. See below.
trajectory[].xintegerRequiredX-coordinate of the pointer at this sample.
trajectory[].yintegerRequiredY-coordinate of the pointer at this sample.
trajectory[].tlongRequiredElapsed time in milliseconds since drag start (first point = 0).
durationintegerRequiredTotal drag duration in milliseconds.

Response — data object

FieldTypeDescription
passedbooleantrue when riskScore ≥ 60.
riskScoreinteger (0–100)Behavioral risk score. Higher = more human-like.
captchaTokenstring | null36-char token issued on pass. Pass to your backend for server-side verification. null on failure.

POST /captcha/token/check
Verify & Consume a Captcha Token
Called by your backend. Atomically verifies and consumes the captchaToken issued by /challenge/submit. This call is idempotent: a consumed token returns valid=true with code TOKEN_CONSUMED, safe to retry.

Request body

FieldTypeRequiredDescription
appIdstringRequiredMust match the appId used when issuing the token.
captchaTokenstringRequiredThe token received from the client via /challenge/submit.

Response — data object

FieldTypeDescription
validbooleantrue when code is SUCCESS or TOKEN_CONSUMED.
codestringToken status. See the status code table below.

Token status codes

codevalidMeaningAction
SUCCESStrueFirst-time consumption. Token is valid.Proceed with business logic.
TOKEN_CONSUMEDtrueToken was already consumed (idempotent retry).Proceed normally — safe to retry.
TOKEN_EXPIREDfalseToken exceeded its 5-minute lifetime.Ask user to retry the captcha.
TOKEN_INVALIDfalseappId mismatch — token exists but belongs to a different app. (Token not found returns TOKEN_EXPIRED.)Check appId config; ask user to retry.

SDK Integration

Web (JavaScript)

Include montnets-captcha-sdk.js in your HTML. The SDK exposes a single global object window.CaptchaSDK. — View source & README on GitHub

HTML + JavaScript
<!-- Step 1: Load the SDK -->
<script src="/montnets-captcha-sdk.js"></script>

<script>
// Step 2: Initialise once on page load
CaptchaSDK.init({
  serverUrl: 'https://captcha-api.int-montnets.com',

  // tokenProvider: call YOUR backend — never expose appSecret in the browser
  tokenProvider: async () => {
    const res = await fetch('/api/captcha-token');
    return (await res.json()).accessToken;
  },

  // onSuccess: receives captchaToken after the user passes
  onSuccess: (captchaToken) => {
    // Forward to your backend together with login credentials
    myServer.login(username, password, captchaToken);
  }
});

// Optional: pre-fetch token to eliminate open latency
CaptchaSDK.warmup();

// Step 3: Trigger from your button
document.getElementById('btn-login').onclick = () => CaptchaSDK.show();
</script>

CaptchaSDK API

MethodDescription
CaptchaSDK.init(config)Initialise. Must be called once before any other method.
CaptchaSDK.show()Open the captcha modal.
CaptchaSDK.close()Programmatically close the modal.
CaptchaSDK.warmup()Pre-fetch and cache the accessToken (returns Promise).
CaptchaSDK.clearCache()Clear locally cached accessToken.

Android (Kotlin)

Copy MontnetsCaptcha.kt and captcha_bridge.html into your project. — View source & README on GitHub

📋
Add <uses-permission android:name="android.permission.INTERNET"/> to your AndroidManifest.xml. Place captcha_bridge.html in app/src/main/assets/.
Kotlin — MainActivity.kt
MontnetsCaptcha.show(
    activity  = this,
    serverUrl = "https://captcha-api.int-montnets.com",

    // tokenProvider: call YOUR backend — never put appSecret in the app
    tokenProvider = { callback ->
        MyServer.getCaptchaToken(
            onResult = { token -> callback(token, null) },
            onError  = { err   -> callback(null, err.message) }
        )
    },

    onSuccess = { captchaToken ->
        // Forward to your backend for server-side verification
        MyServer.login(username, password, captchaToken)
    },

    onError = { message ->
        Toast.makeText(this, "Captcha error: $message", Toast.LENGTH_SHORT).show()
    }
)

iOS (Swift)

Copy MontnetsCaptcha.swift and captcha_bridge.html into your Xcode project. Ensure captcha_bridge.html is listed under Build Phases → Copy Bundle Resources. — View source & README on GitHub

MontnetsCaptcha.show(

    }
)

Risk Engine Rules

The risk engine assigns a score from 0 to 100 based on multiple behavioral dimensions. A score of 60 or above results in a captchaToken being issued.

Score rangeResultcaptchaToken issued
≥ 60PASSYes
40 – 59FAILNo — user may retry
< 40BLOCKNo — strongly automated

Scoring dimensions

DimensionNormal rangeAnomaly signal
Trajectory point count50 – 300 points< 50 (too few) or > 300 (too many / replay)
Total drag duration300 ms – 8000 ms< 300 ms (bot fast-fill) or > 8 s (scripted pause)
Velocity variance≥ 0.0001Constant velocity (linear interpolation) → suspected bot
Jitter (micro-tremor)PresentPerfectly smooth trajectory → suspected bot
Pause count1 – 3 pauses0 pauses (inhuman) or > 3 (scripted hesitation)
Coordinate accuracy|endX − realX| ≤ 5 pxExceeds tolerance → immediate BLOCK, score = 0
ℹ️
Coordinate tolerance and score thresholds are fixed server-side and cannot be adjusted per application. Contact support if you observe false-positive rates that affect your users.
